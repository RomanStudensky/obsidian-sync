Уровень кэша - участок памяти, где временно лежат результаты тяжелых запросов
По скорости опережает БД
Обычно лежит в оперативной памяти

Нюансы:
- лучше использовать при частом чтении и редкой записи;
- использовать срок действия данных;
- нужно поднимать несколько кэшей, чтобы один не стал единой точкой отказа;
- нужно синхронизировать кэши и БД;
- нужно чистить кэш от старых и давно неиспользуемых данных(LFU) или по методом FIFO.
