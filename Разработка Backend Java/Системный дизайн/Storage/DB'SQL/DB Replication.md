Репликация данных - создание нескольких копий БД.
Обычно работают по моделе ведущие/ведомые. Чаще всего один ведомый для записи и несколько ведущих для чтения.
Преимущества:
- повышенная производительность - большее количество ведомых копий быстрее отдают данные из-за распределения нагрузки;
- надежность - если на одной из БД данные будут утрачены физически из-за инцидента, то остальные экземпляры сохранят доступность;
- высокая доступность - система продолжит работать даже если одна БД выйдет из строя.
