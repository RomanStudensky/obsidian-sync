
**Read Uncommitted –** разрешает доступ к ещё не зафиксированным результатам выполнения транзакций.

**Read Committed –** результаты других транзакций становятся доступными после их фиксации, т.е. запрещается аномалия грязного чтения. (по умолчанию в PostgreSQL)

**Repeatable Read –** повторное выполнение операций поиска и выборки данных дает такие же результаты, как первое, т.е. запрещаются аномалии грязного и нечеткого чтения.

**Serializable –** последовательное выполнение транзакций.


Фактичекски в PostgreSQL используются 3 последних, т.к. при использовании **Read Uncommitted** он работает как **Read Committed**.

Указать в SQL

``` sql
SET TRANSACTION ISOLATION LEVEL REPEATABLE READ;
```

Так же возможно указать уровень изоляции транзакций для сеанса или для всего сервера.

Транзакция не всегда может выполниться, обеспечивая требуемый уровень изоляции. В таком случае транзакция отклоняется. 

Пример:

Если обе транзакции работают на уровне изоляции **_Serializable_**, то _PostgreSQL_ будет пытаться гарантировать, что результат выполнения транзакций будет аналогичен их последовательному выполнению. Однако, поскольку обе транзакции основываются на одном и том же начальном состоянии баланса (до изменения), возникает конфликт. 

При попытке зафиксировать транзакции _PostgreSQL_ может обнаружить, что выполнение обеих транзакций нарушает требование сериализуемости. В таком случае, одна из транзакций (например, Транзакция A) будет прервана с ошибкой **_SerializationFailure_**, потому что _PostgreSQL_ не сможет обеспечить уровень изоляции **_Serializable_** для обеих транзакций одновременно.
