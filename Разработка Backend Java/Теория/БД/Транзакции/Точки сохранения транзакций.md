Для транзакций, выполняющих объемную работу мб целесообразно использовать оператор _SAVEPOINT._ Состояние данных сохраняется на момент выполнения оператора. В случае если при продолжении выполнения транзакции обнаруживается ошибка, можно выполнить операцию _ROLLBACK_ с указанием имени того состояния, которое было предварительно записано _SAVEPOINT_.

Пример

```sql
BEGIN TRANSACTION;
BEGIN

SELECT 
FROM aircrafts
WHERE aircraft_code = '320';

UPDATE** **aircrafts
SET range = 6200
WHERE aircraft_code = '320';

SELECT 
FROM aircrafts
WHERE aircraft_code = '320';

SAVEPOINT svp;
SAVEPOINT

DELETE** **FROM aircrafts
WHERE aircraft_code = '320';

DELETE 1
ROLLBACK TO svp
ROLLBACK

SELECT 
FROM aircrafts
WHERE aircraft_code = '320'

ROLLBACK
ROLLBACK
SELECT 
FROM aircrafts
WHERE aircraft_code = '320'

```
