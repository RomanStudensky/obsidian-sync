Позволяет быстро разворачивать автономные приложения.

**Особенности:**
- Автономное Spring приложение
- Встроенный *Tomcat*, *Jetty* или *Undertow* сервер (не нужно деплоить WAR файлы)
- Предоставляет совместимый стартовый набор зависимостей для упрощения конфигурации
- Автоматически настраивает *Spring* и сторонние библиотеки, по возможности
- Предоставляет готовые к работе функции: метрики, *health check* и внешнюю конфигурацию

**Никакой генерации кода и без *XML***
- **Не нужен *XML*-конфиг**  
    Вместо `applicationContext.xml` — аннотации и *Java*-конфиг:  
    `@SpringBootApplication`, `@Configuration`, `@Bean`, `@Component` и т.д., плюс `application.yml/properties`.
    
- **Не требуется генерация исходников/байткода на этапе сборки**  
    *Boot* сам подтягивает и настраивает бины через **автоконфигурацию** (`spring-boot-autoconfigure`) с условиями типа `@ConditionalOnClass`, без запуска внешних генераторов кода или шаблонов.  
    (Динамические прокси *JDK/CGLIB* — это рантайм-механизм, а не “кодогенерация проекта”.)
    
- **“Заводится из коробки”**  
    Стартеры + автоконфиг = минимум явной конфигурации. Например, добавили `spring-boot-starter-web` — получили embedded *Tomcat, MVC, Jackson* и т.п. без *XML*.
    
- **Примечание про AOT/Native** (Boot 3+)  
    [[AOT]]-обработка для *[[GraalVM]]* генерирует вспомогательные метаданные, но это **не** требование писать *XML* или использовать генераторы кода в вашем проекте.
