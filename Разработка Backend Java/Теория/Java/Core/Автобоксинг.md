
Автобоксинг в Java — это автоматическое преобразование между примитивными типами (`int`, `double`, `boolean` и т.д.) и их объектными обёртками (`Integer`, `Double`, `Boolean` и др.).

### Как работает

- **Автобоксинг**: примитив → объектная обёртка.
``` java
int a = 5; Integer b = a; // компилятор автоматически сделает Integer.valueOf(a)
```

- **Автораспаковка (unboxing)**: объектная обёртка → примитив.
``` java
Integer x = 10; int y = x; // компилятор автоматически вызовет x.intValue()`
```

### Где используется

1. **Коллекции** — в `ArrayList`, `HashMap` и других нельзя хранить примитивы, только объекты.

``` java
List<Integer> list = new ArrayList<>(); 
list.add(1); // int → Integer 

int n = list.get(0); // Integer → int`
```

2. **Операции с обёртками**:

``` java
Integer a = 100; 
Integer b = 100; 
System.out.println(a == b); // true (кэш Integer от -128 до 127)  

Integer c = 200; 
Integer d = 200; 
System.out.println(c == d); // false (разные объекты)
```

### Подводные камни

- Возможны `NullPointerException`:

`Integer num = null; int x = num; // NPE при unboxing`

- Производительность: автобоксинг создаёт лишние объекты.  
    В горячих циклах лучше работать с примитивами.
