Наиболее распространенная реализация *JVM* 

**HotSpot** — **основная виртуальная машина Java (JVM) как для клиентских, так и для серверных компьютеров, выпускаемая корпорацией «Oracle»**.  [1](https://ru.wikipedia.org/wiki/HotSpot)

Для повышения производительности обладает технологиями динамической компиляции JIT и адаптивной оптимизации. [1](https://ru.wikipedia.org/wiki/HotSpot)

**HotSpot называется так, потому что, выполняя байт-код Java, она ищет его «горячие» места** — многократно выполняющиеся. Поиск направлен на оптимизацию их выполнения: выделение им больших ресурсов вместе с уменьшением непроизводительных затрат для выполнения менее ресурсоёмкого кода. [1](https://ru.wikipedia.org/wiki/HotSpot)

**HotSpot предоставляет следующую функциональность**: загрузчик Java-классов, интерпретатор байткода, две версии виртуальной машины — Client и Server, несколько сборщиков мусора, набор вспомогательных библиотек времени выполнения. [1](https://ru.wikipedia.org/wiki/HotSpot)

**HotSpot работает в двух режимах**: Client JVM и Server JVM. Client JVM предназначена для запуска небольших клиентских программ на Java, а Server JVM — для запуска больших серверных программ на Java.
