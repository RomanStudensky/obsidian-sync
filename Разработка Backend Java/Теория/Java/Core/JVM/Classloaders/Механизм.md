## Как работает механизм загрузки классов

### Ключевые шаги процесса

**Загрузка (Loading)**  
На этапе загрузки класс-загрузчик находит нужный файл `.class` (или другой источник) и читает его байт-код. После загрузки в памяти создаётся некая структура данных, которая описывает класс (метаданные, информацию о методах, полях и т.д.).

**Связывание (Linking)**  
Этап связывания включает в себя:
- **Проверку (Verification)**: Проверяется корректность байт-кода, соблюдение спецификации языка, целостность и т.п.
- **Подготовку (Preparation)**: Выделяется память для статических переменных, инициализируются значениями по умолчанию.
- **Разрешение (Resolution)**: JVM проверяет и связывает все используемые ссылки на другие классы, методы, поля.
 
**Инициализация (Initialization)**  
Выполняются статические блоки инициализации (static initializers) и присваиваются конечные значения статическим переменным.


Только после прохождения этих этапов класс считается полностью загруженным и готов к использованию в JVM.
