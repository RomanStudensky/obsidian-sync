## Иерархия загрузчиков классов

В Java существует несколько типов «стандартных» загрузчиков, работающих каскадно:

 **Bootstrap ClassLoader**
- Встроен в JVM (написан на нативном коде).
- Занимается загрузкой самых базовых классов Java (например, классов из `rt.jar` или из модуля `java.base` в современных версиях).
- К нему напрямую обраться нельзя, он не является объектом `ClassLoader` на Java уровне.

**Extension ClassLoader (Platform ClassLoader)**
- Загружает классы из стандартных расширений (Java Extensions).
- В современных версиях Java называется Platform ClassLoader и отвечает за загрузку библиотек из системных путей (например, классы в каталоге `lib/ext` в старых реализациях, а также модули платформы).
- Является обычным Java-объектом и расширяет `ClassLoader`.

**Application ClassLoader (System ClassLoader)**
- Занимается загрузкой классов приложения из *classpath* (или *modulepath*).
- Является потомком Platform ClassLoader. По умолчанию именно он отвечает за загрузку пользовательских классов из пути, указанного при запуске приложения.

**Пользовательские ClassLoader-ы**
- Можно создавать собственные загрузчики, унаследовав класс `ClassLoader` или используя `URLClassLoader` (до Java 9) или аналоги.
- Их можно использовать для особых случаев: загрузка из нестандартных источников, «горячее» обновление кода во время работы и т.д.
