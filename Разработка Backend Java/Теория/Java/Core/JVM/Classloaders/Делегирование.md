## Делегирование ClassLoader-а

Традиционно Java использует модель делегирования при загрузке классов:

- Сначала ClassLoader запрашивает у своего «родителя» (parent) загрузить класс.
- Если родитель не находит класс, то уже текущий ClassLoader пытается сам его загрузить.

Данная схема позволяет:

- Исключить дублирование и конфликт версий: если класс доступен на более высоком уровне (например, в базовой библиотеке Java), то он не будет загружен повторно на уровне пользовательского ClassLoader-а.
- Обеспечить безопасность: системные классы всегда загружаются «сверху» из доверенного источника.
