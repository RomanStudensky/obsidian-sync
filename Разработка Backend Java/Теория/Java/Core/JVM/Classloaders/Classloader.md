
Classloader - механизм, позволяющий динамически загружать в JVM классы во время выполнения приложения. Он находит .class файлы или другие источники байт-кода (JAR-архивы, удалённые ресурсы по сети и т.д.) и загружает их в JVM.


## Зачем нужен ClassLoader

**Динамическая загрузка**  
Java позволяет загружать классы «на лету», когда они впервые нужны программе. Это даёт гибкость: классы не обязательно должны быть загружены заранее (в отличие от статической компоновки в некоторых языках).
Динамическая инициализация в Java — это фундаментальный механизм, обеспечивающий:

1. **Гибкость** — классы подгружаются только когда нужны.
2. **Безопасность** — контроль и проверка на этапе загрузки.
3. **Расширяемость** — модульная архитектура, системы плагинов и пр.
4. **Удобство** — поддержка рефлексии, горячей замены и динамических вызовов.

**Разделение пространств имен**  
Каждый загрузчик классов формирует собственное «пространство имён» для загружаемых классов. Это помогает, например, иметь разные версии одних и тех же библиотек, загруженных разными ClassLoader-ами, что предотвращает конфликты.

**Безопасность**  
Система загрузчиков классов тесно связана с системой безопасности (SecurityManager). Она проверяет, откуда взят байт-код, имеет ли приложение соответствующие права и т.д. Класс-загрузчик может накладывать дополнительные проверки, прежде чем разрешить загрузку класса.

**Расширяемость**  
Java даёт возможность создавать собственные классы-наследники `ClassLoader` для специфических целей: загрузка из базы данных, из сети, шифрованных JAR-ов, и т.д.
