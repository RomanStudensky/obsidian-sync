Внутри лежит **массив бакетов** (нод) *Node<K,V>[] table*

**Бакет** -  *Node<K, V>*, в которой лежат *ключ:значение*, *ссылки на следующий элемент* и *хеш*.

*LoadFactor* - при каком проценте заполнения расширяется мапа (0.75 по умолчанию)

**Основные операции:**

- **`put(key, value)` / `get(key)` / `remove(key)`:**
    - **В лучшем:** O(1) – хеш ключа указывает на бакет, поиск там почти мгновенный.
    - **В среднем:** O(log(n)) – когда в бакете дерево.
    - **В худшем:** O(n) – все ключи хэшируются в один бакет, превращая поиск в линейный проход по списку.
- **`containsKey(key)`:** Тоже O(1) в среднем, O(n) в худшем.
- **`size()`:** O(1) (хранит счетчик).
- **`clear()`:** O(1) (сброс ссылок).
